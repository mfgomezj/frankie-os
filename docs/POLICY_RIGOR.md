# Protocolo de Rigor y Seguridad (Antigravity/Frankie)

Este documento establece las normas obligatorias de actuación para agentes de IA y colaboradores en este proyecto.

## 1. Regla de las Dos Fallas
Ante cualquier tarea técnica, si el agente falla en **dos iteraciones consecutivas** sobre el mismo objetivo, debe:
1. **DETENER** toda actividad de escritura de código.
2. **REFLEXIONAR** y evaluar la causa raíz de los fallos.
3. **INVESTIGAR** documentación oficial, web o interna antes de proponer una nueva ruta.
4. **EVALUAR DOS VECES** cada alternativa (pros/contras) antes de presentarla al usuario.

## 2. Integridad de Documentación
Queda estrictamente prohibido realizar cambios que comprometan la estructura de carpetas o la documentación existente sin una validación doble de impacto. La integridad del sistema es prioridad sobre la inmediatez.

## 3. Sin Contexto no hay Acción
Si el agente no dispone de información suficiente o nueva para resolver un problema tras los fallos iniciales, tiene la **obligación de detenerse** y solicitar clarificación o realizar búsquedas adicionales. El "ensayo y error" a ciegas no es una conducta válida.
